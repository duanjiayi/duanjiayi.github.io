How to build portfolio website

记录我通过Hexo+GitHub搭建个人设计作品网站的过程
通过在国内国外调研了一波，阿里云、Wordpress、Squarespace、GitHub Pages等，最终选择Hexo+GitHub。

Hexo

1. 🆓 免费

比较完阿里云，Wordpress的价格，请捂住荷包看看hexo

2. 🎨 提供297种不同风格的主题

基于主题用户可以通过配置文件完成样式的配置，没有前端基础仔细阅读Read.me也可以轻松搞定

3. 💻 简洁，易上手

  • 安装只需要几步代码，非常简单
  • 除官网自带的安装部署教程外，网上有现成的非常多的教程和案例，另外遇到的问题在Google/Baidu搜索基本可以找到答案

4. ✍️ 支持Markdown写作

我们搭建个人作品网站的目的是去展示我们的作品，所以框架服务于内容，Markdown对写作格式的友好没得说，太ok了

5. 📱Hexo支持响应式

手机/iPad也可以有友好地访问体验


GitHub

方案初期选择将网站部署到GitHub上，主要原因是：免费
备注：在国内访问速度稍微有点慢


如何搭建？

除官方教程外，网上也有一些更加详细的教程，我推荐这篇文章《hexo史上最全搭建教程》，看完包教包会


常见问题

1. 如何购买域名？

  1. 万网
  2. GoDaddy
  3. 短期内如果没有购买域名需求，可以通过部署到GitHub Pages，也完全免费

2. 没有代码基础是否可以完成？

我是一个UX Designer，由于工作需要，前端html/css可以略看懂，但是不具备实操能力，我个人认为操作起来完全没问题。
整体操作基本不需要写代码，当然如果你有代码基础，也可以自己来自定义。

3. 搭建一个网站大概需要多久？

以我为例按照教程操作，2-3小时基本可以搞定

4. 如何在Markdown中插入图片？

我是用的图床方式，推荐iPic，免费🆓


写在最后

任何框架都是为内容服务，高质量的内容输出才是重点，最后希望大家都可以顺利完成作品网站的搭建✌️
